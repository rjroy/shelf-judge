**MEETING NOTES — Previously Owned Feature: Filter System Integration**

Sienna identified that the "Show Previously Owned" toggle button was overly prominent and questioned its placement in the filter bar. The toggle was also redundant with the filter system's purpose—it controls which records appear, making it functionally a filter rather than a standalone control. Discussion centered on two problems: visual clutter (the button is large) and discoverability (users with no previously-owned games see an irrelevant control).

Agreement reached on moving the toggle into the Filters panel rather than displaying it as a top-level button. The control only displays when previously-owned games exist (previouslyOwnedCount > 0). When active, it generates a filter chip ("Prev Owned ×") and the Filters button carries a numeric indicator (e.g., badge "1" or dot) to signal the filter is active or that hidden games exist. Discovery of the toggle on first use is handled via toast/confirmation messaging on the game detail page when the user marks a game as previously owned.

Mockup updated to show three states: (A) clean filter bar with no previously-owned toggle, (B) Filters panel expanded showing "Owned Status" group as one of several filter groups, (C) filter active with chip and Filters button badge. Pattern matches existing filter/sort UX in the mockup system.

**Decision.** Previously-owned visibility control moves from standalone button to Filters panel. Conditional display based on inventory. Discovery path managed through game detail page confirmation, not filter bar visibility.

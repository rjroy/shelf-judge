Design Decisions

Provenance color language

Amber = personal ratings and weights (your data). Slate blue = BGG-derived (external data). Navy = navigation/actions. This distinction is critical throughout: BGG attribute bars are always blue, histogram bars for personal ratings are always amber.

Micro-histograms for distributions

Mean/median/SD stats alone don't show shape. A bimodal distribution (Strategic Depth) looks identical to a flat one in a table. The 10-bucket histogram at 36px tall fits inline per axis and makes clustering and bimodality visible at a glance.

Divergence as score comparison

Showing both fitness and tournament scores side-by-side makes the tension legible without explanatory text. The directional gap tag (▲ T = tournament higher, ▲ F = fitness higher) names which way the divergence runs without requiring the user to do mental subtraction.

Outlier component transparency

The spec requires per-component distances to be available (REQ-PROFILE-11). Showing them as small chips below each outlier game answers the user's first question: "why?" High-distance components are highlighted purple; low ones read as context.

Axis suggestions as questions

Spec (REQ-PROFILE-16) requires suggestions as questions, not imperatives. The color dot encodes source type without adding a label field. Dismiss is session-only per spec — no persistence, no "never show again."

LLM narration as optional top slot

Narration is deferred post-MVP but the slot is always present so users see what's coming. Empty state shows a Generate button. Stale state keeps the old narration visible (not hidden) per REQ-PROFILE-27 — stale narration is better than nothing.
